Managing containers is very complexe and challenging because it obeys to many rules susch as the DNS configuration, login to the registry, retrieve the right iso image, pull the iso image and so on.
In this project, we have launched 8 containers starting from basic to advanced and have configured all of them to automatically start at the boot or reboot.
In the first time, we have listed and have performed all requierments for container launching by installing the container software, by loging in to the registry, and by retrieving the iso image.
In the second time, we have launched a basic container using the /rhel8/rsyslog iso image and have configured it to automatically start at the boot or reboot. In the same way, we have launched another basic container using /rhel7/ubi7 iso image.
In the third time, we have lauched a container with a specific port and have configured it to automatically start at any boot or reboot. In addition, we have accessed the container and have ran fews services.
In the fourth time, we have launched a container with an environment variable and a variable and also have configured it to auto start at the boot.
In the fifth time, we have launched a container with an existing user and have configured to auto start at the boot or reboot.
Finally, we have launched a container with a persistent storage and have configured it to auto start at the boot or reboot.
